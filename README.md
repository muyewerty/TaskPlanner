# TaskPlanner
Проект разработан для эффективного управления личным временем, структурирования списков дел и контроля за дедлайнами с помощью системы нативных уведомлений.

# Функции

При разработке были разделены пользовательские функции и архитектурные решения. 

**Функционал приложения:**

* **Полное управление задачами (CRUD): Быстрое создание, редактирование и удаление задач.**

* **Гибкая система напоминаний: Возможность установить таймер из готового списка (1, 5, 10 минут) или задать собственное время через кастомный выпадающий список (CustomSpinner). **

* **Удобная навигация: Просмотр списка задач с прокруткой и мгновенным переключением между главным экраном и экраном редактирования.**

# 🌟 Особенности (архитектура):

* **Автономность (Офлайн-работа): Проект не привязан к облачным сервисам. Все пользовательские данные сохраняются локально в легковесном формате JSON, что обеспечивает быстрый доступ к задачам без интернета.**

* **Кроссплатформенность: Благодаря аппаратному ускорению (OpenGL) и возможностям фреймворка Kivy, приложение одинаково хорошо работает на Windows, macOS и Linux, и готово к портированию на мобильные устройства.**
  
* **Асинхронность интерфейса: Таймеры напоминаний работают независимо от графической оболочки, поэтому приложение никогда не «зависает» в ожидании дедлайна.**

# 🛠 Технологии

Для реализации проекта был выбран стек на базе Python, так как он позволяет сочетать скорость разработки и кроссплатформенность.

* **Python 3.x** — выбран как основной язык из-за богатой экосистемы библиотек и высокой читаемости кода, что упрощает поддержку и расширение функционала.

* **Kivy** — этот фреймворк использован для построения GUI, так как он поддерживает **GPU-ускорение** и позволяет собрать приложение под Windows, Linux, macOS и даже Android/iOS без переписывания логики.

* **Plyer** — необходим для реализации кроссплатформенных уведомлений. Вместо написания уникального кода под каждую ОС, Plyer предоставляет универсальный интерфейс для обращения к системному API.

**Вспомогательные модули:**

* **`json`**: Выбран как альтернатива полноценным БД (вроде SQLite). Для компактного планировщика это идеальный формат: данные легко читаются человеком, не требуют сервера и моментально загружаются в память.

* **`os`**: Критически важен для работы с путями. Гарантирует, что программа корректно найдет `tasks.json` вне зависимости от того, запущена она из IDE или как собранный `.exe` файл.

* **`threading` (Многопоточность)**: В Kivy основной поток занят отрисовкой интерфейса. Использование `threading` позволяет вынести ожидание таймера в отдельный поток. 

* **`time`**: Используется для управления задержками в потоках напоминаний, обеспечивая точность срабатывания уведомлений.
